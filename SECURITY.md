# Security Policy

## Supported Versions

Only the latest `main` branch is considered supported for security fixes.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Preferred options:

1. Use GitHub's private vulnerability reporting or security advisory flow if it is enabled for this repository.
2. If private reporting is not available, open a normal issue with a short summary and mark it clearly as security-sensitive without sharing exploit details.

## What to Include

- A short description of the issue
- The affected file, feature, or flow
- Steps to reproduce, if safe to share
- Any mitigation you recommend

We will review reports as soon as possible and coordinate a fix privately when needed.
